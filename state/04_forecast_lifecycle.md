# Purpose

This document defines the authoritative lifecycle for Forecast entities.

All forecasting workflows must follow this lifecycle.

No implementation may violate these transitions.

---

# Lifecycle Overview

```text
Draft
 ↓
Pending Review
 ↓
Pending Approval
 ↓
Published
 ↓
Archived
```

---

# Lifecycle Principles

Forecasts are:

```text
Advisory
Explainable
Auditable
Human Approved
```

Forecasts may not:

```text
Automatically Modify Workforce Plans
Automatically Create Positions
Automatically Create Vacancies
Automatically Trigger Personnel Actions
```

---

# State Definitions

## Draft

Description:

Forecast generated by the AI forecasting engine.

Characteristics:

```text
Editable
Reviewable
Not Published
Not Operational
```

---

## Pending Review

Description:

Forecast under workforce planner review.

Characteristics:

```text
Review Active
Comments Allowed
Recommendations Visible
```

---

## Pending Approval

Description:

Forecast submitted for executive approval.

Characteristics:

```text
Read Only
Awaiting Approval
Operational Review Complete
```

---

## Published

Description:

Official workforce planning forecast.

Characteristics:

```text
Visible
Reportable
Historical Baseline Established
```

---

## Archived

Description:

Historical forecast retained for reporting and audit.

Characteristics:

```text
Read Only
Historical
Immutable
```

---

# Allowed Transitions

## Draft → Pending Review

Requirements:

```text
Forecast Generated
Confidence Calculated
Explanation Generated
```

Approval:

```text
System Generated
```

---

## Pending Review → Pending Approval

Requirements:

```text
Planner Review Complete
Comments Recorded
```

Approval:

```text
Workforce Planner
```

---

## Pending Approval → Published

Requirements:

```text
Executive Approval Granted
Forecast Valid
```

Approval:

```text
HR Director
```

---

## Published → Archived

Requirements:

```text
Forecast Superseded
Retention Criteria Met
```

Approval:

```text
System Automated
Administrator
```

---

# Rejection Workflow

Forecasts may be rejected.

Process:

```text
Pending Review
 ↓
Rejected
 ↓
Regenerate Forecast
 ↓
Draft
```

or

```text
Pending Approval
 ↓
Rejected
 ↓
Draft
```

---

# Additional State

## Rejected

Description:

Forecast not approved.

Characteristics:

```text
Read Only
Historical
Requires Regeneration
```

---

# Allowed Rejection Transitions

```text
Pending Review → Rejected
Pending Approval → Rejected
Rejected → Draft
```

---

# Forbidden Transitions

```text
Draft → Published
Draft → Archived
Pending Review → Published
Archived → Published
Archived → Draft
Published → Draft
```

---

# Confidence Rules

Every forecast must contain:

```text
Confidence Score
Forecast Factors
Explanation
Generation Timestamp
```

---

# Explainability Rules

Every forecast must include:

```text
Primary Factors
Narrative Explanation
Confidence
```

Black-box forecasts prohibited.

---

# Governance Rules

Human approval mandatory before publication.

AI may:

```text
Generate Forecast
Analyze Trends
Provide Explanations
```

AI may not:

```text
Publish Forecast
Approve Forecast
Modify Workforce Plans
```

---

# Entry Events

Generated:

```text
Forecast Generated
Forecast Submitted
Forecast Approved
Forecast Published
Forecast Rejected
Forecast Archived
```

---

# Exit Events

Generated:

```text
Forecast State Changed
```

---

# Notification Requirements

Generate notifications for:

```text
Forecast Submitted
Forecast Approved
Forecast Rejected
Forecast Published
```

---

# Audit Requirements

Every transition generates:

```text
Audit Event
Timestamp
Actor
Reason
```

---

# Failure Rules

If validation fails:

```text
Block Transition
Generate Audit Event
Notify User
```

---

# State Integrity Rules

Forecast must exist in exactly one state.

Multiple simultaneous states prohibited.

---

# Acceptance Criteria

Lifecycle is valid when:

1. Human approval enforced.
2. Explainability enforced.
3. Audit logging active.
4. Notifications generated.
5. AI governance enforced.
