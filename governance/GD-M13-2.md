# Context

spec/06_api_contracts.md defines four employee-scoped endpoints for skills and
certifications but does not define catalog management endpoints for creating or
listing skills and certifications as standalone resources. EMP-601 (GD-M12-4)
mandates that M13 deliver Skills catalog CRUD API and Certifications catalog CRUD API.
This decision establishes the authoritative endpoint design, request contracts, RBAC,
and behavioral constraints for the complete M13 API surface. It supplements spec/06
for catalog management endpoints, which are not present in that document.

---

# Decisions

## Decision 1 — Controller Structure

Skills and Certifications catalog management shall be implemented as two separate
NestJS controllers registered in WorkforceModule:

```text
SkillController         — handles /api/v1/skills routes
CertificationController — handles /api/v1/certifications routes
```

Employee-scoped assignment endpoints shall be implemented in EmployeeController:

```text
POST /api/v1/employees/{id}/skills
GET  /api/v1/employees/{id}/skills
POST /api/v1/employees/{id}/certifications
GET  /api/v1/employees/{id}/certifications
```

## Decision 2 — Skills Catalog Endpoints

```text
GET   /api/v1/skills          — list skills (tenant-scoped, paginated)
POST  /api/v1/skills          — create skill
GET   /api/v1/skills/{id}     — get skill by ID
PATCH /api/v1/skills/{id}     — update skill
```

These endpoints are authorized by EMP-601 (GD-M12-4) and supplement spec/06.

## Decision 3 — Certifications Catalog Endpoints

```text
GET   /api/v1/certifications          — list certifications (tenant-scoped, paginated)
POST  /api/v1/certifications          — create certification
GET   /api/v1/certifications/{id}     — get certification by ID
PATCH /api/v1/certifications/{id}     — update certification
```

These endpoints are authorized by EMP-601 (GD-M12-4) and supplement spec/06.

## Decision 4 — POST /api/v1/skills Request Contract

```text
Required: name (string, max 255 characters)
Optional: category (string, max 100 characters)
Optional: description (string, text)
```

tenant_id: derived from JWT; never accepted from request body.

## Decision 5 — POST /api/v1/certifications Request Contract

```text
Required: name (string, max 255 characters)
Required: expirationRequired (boolean)
Optional: issuer (string, max 255 characters)
```

tenant_id: derived from JWT; never accepted from request body.

## Decision 6 — POST /api/v1/employees/{id}/skills Request Contract

```text
Required: skillId (UUID) — must reference an existing skill within the same tenant.
Optional: proficiencyLevel (string) — must be one of the canonical values defined
          in directives/14_skill_management_rules.md if provided.
Optional: verifiedAt (ISO 8601 timestamp) — when null or absent, the assignment is
          self-declared (unverified). When present, the assignment is HR-verified.
          The verifiedAt value is critical for future skill matching weight per
          directives/05_skill_matching_rules.md SKM-402.
```

## Decision 7 — POST /api/v1/employees/{id}/certifications Request Contract

```text
Required: certificationId (UUID) — must reference an existing certification within
          the same tenant.
Optional: status (string) — must be one of the values defined in GD-M13-3.
          When omitted, defaults to ACTIVE per GD-M13-3 Decision 2.
Optional: issueDate (ISO 8601 date).
Optional: expirationDate (ISO 8601 date).
```

When the referenced certification has expirationRequired = true:
  expirationDate is required.
  Omission returns HTTP 422, error code: EXPIRATION_DATE_REQUIRED.

When both issueDate and expirationDate are provided:
  expirationDate must be >= issueDate.
  Violation returns HTTP 422, error code: INVALID_DATE_RANGE.

## Decision 8 — Skill Name Uniqueness

Skill names must be unique within a tenant.

Duplicate skill name within the same tenant:

```text
HTTP 409, error code: SKILL_NAME_CONFLICT
```

## Decision 9 — Certification Name Uniqueness

Certification names must be unique within a tenant.

Duplicate certification name within the same tenant:

```text
HTTP 409, error code: CERTIFICATION_NAME_CONFLICT
```

## Decision 10 — EMP-302 Enforcement on Employee Assignment Endpoints

POST /api/v1/employees/{id}/skills and POST /api/v1/employees/{id}/certifications
must verify that the target employee's employment_status is not SEPARATED before
performing any write operation.

A request targeting a SEPARATED employee:

```text
HTTP 422, error code: EMPLOYEE_SEPARATED
```

This check applies regardless of the requesting user's role. Authority: EMP-302.

## Decision 11 — Upsert Semantics on Assignment Endpoints

Per GD-M13-4, workforce.employee_skills and workforce.employee_certifications use
composite primary keys. Assignment endpoints use upsert semantics:

```text
First assignment:  INSERT — emits ASSIGNED audit event.
Repeat assignment: UPDATE — emits UPDATED audit event.
```

No error is returned for repeat assignments; the record is updated with the new
field values and the appropriate audit event is emitted.

## Decision 12 — RBAC Matrix

```text
POST  /api/v1/skills                              — SA, HR Director
PATCH /api/v1/skills/{id}                         — SA, HR Director
GET   /api/v1/skills                              — SA, HR Director, Workforce Planner, Compliance Officer
GET   /api/v1/skills/{id}                         — SA, HR Director, Workforce Planner, Compliance Officer

POST  /api/v1/certifications                      — SA, HR Director
PATCH /api/v1/certifications/{id}                 — SA, HR Director
GET   /api/v1/certifications                      — SA, HR Director, Workforce Planner, Compliance Officer
GET   /api/v1/certifications/{id}                 — SA, HR Director, Workforce Planner, Compliance Officer

POST  /api/v1/employees/{id}/skills               — SA, HR Director
GET   /api/v1/employees/{id}/skills               — SA, HR Director, Workforce Planner, Compliance Officer

POST  /api/v1/employees/{id}/certifications       — SA, HR Director
GET   /api/v1/employees/{id}/certifications       — SA, HR Director, Workforce Planner, Compliance Officer

Unauthorized for all M13 endpoints: Recruiter, Executive User
Unauthorized response: HTTP 403
```

Compliance Officer read access is required for audit, policy verification, and
FR-153 expiration report generation.

Executive User is excluded from all employee-scoped M13 endpoints per RBAC-952.
RBAC-952 prohibits Executive User access to individual employee records.
Employee skills and certifications are employee-level records.

## Decision 13 — SEC-003 Enforcement

All M13 endpoints derive tenant_id from the JWT exclusively. tenant_id must never
be accepted from a request body, query parameter, or route parameter on any endpoint.
Cross-tenant records return HTTP 404. Response is identical for absent and cross-tenant
records. No tenant enumeration. Authority: GD-M13-1, SEC-003.
