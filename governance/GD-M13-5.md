# Context

spec/04_domain_model.md defines Position → Skills and Position → Certifications
relationships in the Position entity's relationship tree. FR-100 lists Required Skills
and Required Certifications as Position inputs.

However:

```text
spec/05_database_schema.md — no position_skills or position_certifications tables
spec/06_api_contracts.md   — no position skills or certifications endpoints
GD-PRE-M13-002             — position-employee linkage deferred beyond M13
```

Position skill and certification requirements cannot be meaningfully enforced without
the position linkage mechanism that GD-PRE-M13-002 has explicitly deferred. Implementing
position skill/cert tables in M13 would add schema with no enforcement path, no API
authority, and no resolvable dependency on position linkage.

---

# Decisions

## Decision 1

position_skills and position_certifications tables are not created in M13.
No schema migration for these tables is included in the M13 migration.

## Decision 2

No API endpoints for managing position skill or certification requirements are
implemented in M13. This includes, but is not limited to:

```text
POST   /api/v1/positions/{id}/skills
GET    /api/v1/positions/{id}/skills
DELETE /api/v1/positions/{id}/skills/{skillId}
POST   /api/v1/positions/{id}/certifications
GET    /api/v1/positions/{id}/certifications
DELETE /api/v1/positions/{id}/certifications/{certId}
```

## Decision 3 — FR-100 Capability Maturity

FR-100 Required Skills and Required Certifications capability maturity after M13 is:

```text
Planned — requirements defined; relationship defined in spec/04; no schema or API
authority exists in spec/05 or spec/06; no implementation in M13.
```

## Decision 4 — Future Milestone Prerequisites

Position skill and certification requirements shall be addressed at a future milestone
that satisfies ALL of the following prerequisites:

```text
(a) Schema authority: spec/05_database_schema.md must be amended to define
    position_skills and position_certifications table structures before any
    schema migration for these tables is written.

(b) API authority: spec/06_api_contracts.md must be amended to define endpoint
    contracts for position skill and certification management before any controller
    or service is implemented.

(c) Position linkage mechanism: the position-employee linkage resolution required
    by GD-PRE-M13-002 must be implemented and operational. Position skill
    requirements are only enforceable when employees can be linked to positions
    and the enforcement query path exists.

(d) M13 prerequisite: M13 must be complete. The position skill requirements
    milestone depends on workforce.skills(id) and workforce.certifications(id)
    existing and being populated. M13 is a hard prerequisite for this future
    milestone.
```

## Decision 5 — No Provisional Implementation

No provisional or placeholder position_skills or position_certifications tables,
models, or endpoints shall be added to the repository in M13. Adding empty tables
or stub endpoints for a deferred capability creates schema drift and violates the
CLAUDE.md principle that scaffolded code must not be treated as complete.

---

# Rationale

Implementing position skill requirements in M13 would require simultaneous spec
deviations for schema (spec/05) and API (spec/06) with no authoritative design
to follow. The position linkage mechanism (GD-PRE-M13-002) that position skill
enforcement depends on has not been resolved. Proceeding without schema authority,
without API authority, and without the prerequisite dependency would produce an
unenforceable capability fragment rather than a production-ready feature.

Explicit deferral via this governance decision ensures the gap is visible, traceable,
and guarded against premature implementation.

---

# Spec Deviation

None.

This decision follows spec/05 and spec/06 exactly. The deferral is a formal record
of what the specification already implies by omission.
