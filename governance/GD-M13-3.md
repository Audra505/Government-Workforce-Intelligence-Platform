# Context

spec/05_database_schema.md defines status VARCHAR(50) on workforce.employee_certifications
but does not enumerate valid values in any specification document. The status field is the
mechanism for FR-153 expiration tracking and FR-151 certification validity management.
Without canonical values, implementation would produce non-deterministic status
vocabularies across the system. This decision establishes the authoritative set of
status values and their behavioral semantics for M13.

---

# Decisions

## Decision 1 — Canonical Status Values

The following three values are the only permitted values for
workforce.employee_certifications.status in M13.

```text
ACTIVE   — The certification is currently held and within its validity period.
            This is the normal state for a current, non-expired certification.

EXPIRED  — The certification's expirationDate has passed and the certification
            is no longer valid. This status is the mechanism for FR-153 expiration
            tracking queries. May be set by HR explicitly or by a future system
            process. An ACTIVE certification whose expirationDate is in the past
            should surface in expiration tracking queries regardless of its stored
            status value (queries must check both status field and date value).

REVOKED  — The issuing authority has invalidated the certification before its
            natural expiration date. This is always an explicit HR-managed state.
            It cannot be derived from date fields. REVOKED is terminal for M13:
            no outbound transitions from REVOKED are permitted.
```

No other status values are permitted in M13. Any status value not in this set
must be rejected at the DTO validation layer with HTTP 422,
error code: INVALID_CERTIFICATION_STATUS.

## Decision 2 — Default Status on Assignment

When the status field is omitted from POST /api/v1/employees/{id}/certifications,
the system automatically assigns ACTIVE as the status value.

The caller need not provide status for a standard new certification assignment.
When status is explicitly provided, it must be one of the three canonical values.
An invalid status value returns HTTP 422, error code: INVALID_CERTIFICATION_STATUS.

## Decision 3 — Allowed Status Transitions

The following transitions are permitted via the assignment endpoint
(upsert update semantics per GD-M13-4 Decision 3):

```text
ACTIVE  → EXPIRED  (HR-managed; or future system process detecting past expirationDate)
ACTIVE  → REVOKED  (HR-managed only; explicit action required)
EXPIRED → ACTIVE   (HR-managed; used when a certification is renewed — the record is
                     updated with new issueDate, new expirationDate, status ACTIVE;
                     generates WORKFORCE_EMPLOYEE_CERT_RENEWED audit event)

REVOKED → (none; REVOKED is terminal for M13)
```

## Decision 4 — Expiration Tracking Behavior

The GET /api/v1/employees/{id}/certifications endpoint must include the expirationDate
and status fields in its response for each certification assignment.

Expiration tracking queries (FR-153) must surface certifications where:

```text
status is ACTIVE
AND expirationDate is not null
AND expirationDate is within the requested lookahead window (within N days)
```

Certifications with status EXPIRED or REVOKED are not surfaced as "expiring" — they
are already expired or invalidated.

## Decision 5 — Matching Compatibility

EXPIRED and REVOKED statuses satisfy directives/05_skill_matching_rules.md SKM-302:
"Expired certifications treated as missing."

The Phase 3 AI matching engine must treat employee certifications with status EXPIRED
or REVOKED as non-qualifying for required certification checks. This behavioral contract
is established here so that M13 implementation does not inadvertently create a data
model that conflicts with Phase 3 matching requirements.

## Decision 6 — Deferred Status Values

SUSPENDED and PENDING_VERIFICATION are not included in M13. Any addition of status
values beyond ACTIVE, EXPIRED, and REVOKED requires a new governance decision before
implementation.
